# 内容发布日历

用这个模板，固定每周发布 1 篇“可复现、可执行”的实战文章。

## 每周节奏

1. 周一：确定一个真实用户痛点
2. 周二：收集可复现命令和截图
3. 周三：起草文章（问题 -> 命令 -> 验证 -> 坑点）
4. 周四：内部技术评审
5. 周五：发布并同步到更新日志/社区

## 文章模板

```markdown
# [结果导向标题]

## 问题
- 谁被卡住了？
- 真实场景里哪里失败？

## 复现
```bash
# 最小命令
```

## 修复模式
```bash
# 稳定命令序列
```

## 验证
- 用什么命令证明成功？
- 用什么产物证明成功？

## 常见坑
- 典型配置错误
- 站点/provider 波动说明
```

## 建议 4 周主题

1. 稳定 smoke 与动态站点 smoke 的分层策略
2. 一键 onboarding 与失败恢复
3. provider 路由的成本/性能策略
4. harness checkpoint 驱动的长任务重构
